# Moto Flash Express Non Funtional Requirements

- Data Security: All sensitive information (payments, profiles) must be encrypted under the AES-256 standard.

- Availability: The system must guarantee an uptime of 99.9%.

- Scalability: The architecture must allow for growth from 3 to 5 motorcycles or more without service degradation.

- Response Time: Application screens must load in less than 2 seconds.

- Usability: The interface must be intuitive for older adults or individuals with limited digital experience.

- Localization: The system must be specifically configured for the Ameca and Valles de Jalisco region.

- Reliability: The system should maintain a failure history near zero to preserve client trust.

- Portability: The mobile application must be compatible with Android and iOS systems.

- Maintainability: The code must be documented to facilitate future updates (Digitalization Phase).

- Legal Restrictions: The system must comply with local traffic regulations and privacy policies.
